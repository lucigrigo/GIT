Nivelul fizic = nivelul de baza atat in stiva OSI, cat si in stiva TCP/IP. Se ocupa in principal cu transmiterea bitilor de la o statie la alta la nivel fizic, adica prin diferite metode in functie de mijlocul de comunicare ales: lumina in fibra optica, unde electromagnetice in aer, etc. Nu interpreteaza pachete, datagrame, sau alte forme mai avansate de reprezentare a datelor. Mai are rol in: sincronizarea bitilor, codificare/decodificare, modulatie, etc.
Codificare Manchester: in mijlocul perioadei de ceas: tranzitie de jos in sus = 1, tranzitie de sus in jos = 0. Tranzitiile de la inceputul perioadei de ceas sunt folosite pentru sincronizare.

2.
DHCP = Dynamic Host Configuration Protocol
Folosit in medii unde configurarea unei adrese unui host se face de catre un server, locatii publice precum un aeroport. Etape:
a) DHCP_DISCOVER - hostul transmite broadcast un mesaj: "exista vreun server DHCP in zona?"
b) DHCP_OFFER - serverul DHCP ofera hostului o "oferta" pentru o adresa IP pe care o poate folosi in Internet
c) DHCP_REQUEST - hostul cere serverului DHCP sa ii ofere acea adresa
d) DHCP_ACK - serverul confirma hostului ca poate folosi adresa oferita

NAT si PAT
NAT = Network Address Translation - mijloc de comunicare intre o retea locala de hosturi si restul Internetului. Dpdv exterior, reteaua este reprezentata printr-o singura adresa, cea a serverului de NAT. Din interior, orice host care doreste sa trimita ceva in exterior trebuie sa treaca pachetele prin serverul de NAT. +securitate (din exterior nu se pot vedea in mod direct hosturile) +exista alocate anumite intervale pentru adrese care in retea sunt atribuite unic, dar in Internet nu sunt unice. -dificultati atunci cand se doreste conectarea in mod direct catre un host dintr-o retea cu NAT (exemplu apel Skype catre un server dintr-un NAT -> folosirea Skype Relay)
PAT = Port Address Translation - functioneaza similar cu NAT, dar identificarea la nivel de server PAT a hosturilor pe care doreste sa le translateze are loc la nivel de port, unde fiecare primeste unul in mod unic. -si NAT si PAT inlocuiesc adresele IP sursa. -dezavantaje cand trebuie calculata o cale care trece printr-un server de PAT sau NAT

CSMA/CA - folosit in comunicarea de tip acces multiplu. Este un algoritm urmat de statiile dintr-un astfel de sistem, care merg pe principiul ca mai intai asculta si dupa transmit ("polite conversationalist"). Astfel, se asigura ca mediul de comunicare nu este ocupat pentru o anumita perioada de timp (delay de trimitere/primire) si dupa pot transmite. De asemenea, statiile memoreaza si un timer de la ultima incercare a vedea daca reteaua este libera.
